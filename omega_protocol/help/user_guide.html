OMEGA Protocol

User Guide

OMEGA Protocol is a Windows data sanitization tool designed to make secure deletion easier to understand and easier to control. The app focuses on practical workflows, clear operator feedback and honest results.

What each area does

Files

Add, review and remove file targets for file-level sanitization.

Drives

Shows available storage targets. In limited mode the app may show logical volumes for visibility while waiting for administrator access.

Preflight

Shows what the tool plans to do before anything destructive happens.

Report

Displays the final session summary and the report files created for that run.

What the buttons do

  • Add Files: add one or more files to the current session.
  • Remove Selected Files: remove selected file entries from the queue.
  • Refresh Drives: reload drive inventory and update the preflight view.
  • Clear Selection: remove the current selection from the session.
  • Run Session: start the selected workflow after confirmation when required.
  • Help: open this user guide in your browser.

What Dry Run means

Dry Run lets you preview the workflow without making destructive changes. It is useful when you want to confirm the targets, check warnings and understand the expected method before real execution.

Dry Run is ideal for validation, training, demonstrations and final checks before a real sanitize action.

What the tool does to reduce recovery risk

  • Validates paths and blocks risky path types such as network targets or reparse points.
  • Uses a layered overwrite-and-remove workflow for supported file targets.
  • Uses device-oriented workflows when Windows and the storage stack expose them.
  • Creates auditable reports so each run can be reviewed later.
  • Downgrades results instead of pretending a stronger erase level was achieved.

Why SSD results can vary

SSD media uses internal management techniques that can move data away from the exact filesystem location a user can see. Because of that, file-level workflows on SSD media are presented as best effort.

For whole-drive workflows, OMEGA Protocol prefers device-oriented methods when the operating system and the hardware expose them. That path can be much stronger than a normal file delete.

Why other media can be simpler

On some rotational drives and on some controller paths, the storage stack can expose a clearer relationship between the request and the physical media action. In those cases OMEGA Protocol can report a stronger result, but only after the system actually accepts the requested workflow.
